how does javascript work?

javascript always works line by line,as it is a single threaded language,It has only one stack so it couldnot run multiple executions at the same time, Basically javascript works synchronously ,that means it executes in top to bottom order,but for blocked code execution which are dependent on other codes, it becomes asynchronous,means it keeps executing other non blocked codes till it gets the reponse of blocked codes which are dependent,that makes it more faster,it needs to be interpreted or converted to machine language to interact with the computer,It's dynamic typing system does not require declaring the vairable types,so it's type cannot be known before the codes are executed

local storage vs session storage

local storage and session storage both are used to store data in browser,the only difference is session storage deletes the data in every session, that means the session storage will be empty for every new tab,So whenever we close a tab or a page all the session storage data will be lost,And even if we create a new tab of the same site that will create another session for that site, that is not linked with the previous session,but local storage doesnot delete it's data till it is explicitly deleted, or forcibly deleted by the user,So if we close the tab or even the machine it will not delete the items stored in it,And if we start a new tab of the same site it will keep the data of the site in that new tab also,So when we try to save some data for e longer period of time we should use local storage, and if we want the data for temporary time we should use session storage
